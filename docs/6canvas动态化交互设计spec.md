# 🔮 AI算命小屋 · Canvas 动态化交互设计 Spec

> **设计哲学**：占卜的本质是"仪式感"。所有动效与交互都服务于三个目的——**沉浸（让人想留下）、暗示（让人知道哪里能点）、戏剧（让占卜结果有"被揭晓"的爽感）**。

---

## 一、用户体验主线（先想清楚 Why，再谈 How）

### 1.1 用户旅程（User Journey）

```
进入 ─→ 环境苏醒 ─→ 视线聚焦 ─→ 探索暗示 ─→ 主动占卜 ─→ 仪式等待 ─→ 结果揭晓 ─→ 二次互动
 0s     0~1.5s       1.5~3s      3s~        点击触发     1.5~3s      高潮       分享/再卜
```

| 阶段         | 用户感受         | 设计目标                       |
| ------------ | ---------------- | ------------------------------ |
| **环境苏醒** | "哇，这地方活的" | 霓虹依次点亮，建立场域         |
| **视线聚焦** | 自然看向中心     | 水晶球最亮、有呼吸光晕         |
| **探索暗示** | "这些好像能点？" | 可交互元素有微妙 idle 动效     |
| **主动占卜** | "我要试试"       | hover 反馈明确、点击有"重量感" |
| **仪式等待** | 紧张期待         | 占卜中要有"在思考"的过程感     |
| **结果揭晓** | 惊喜 / 共鸣      | 卡片翻转 + 粒子爆发 + 文字浮现 |
| **二次互动** | 想分享或再来一次 | 提供"重抽""分享""保存"出口     |

### 1.2 设计铁律

1. **节奏分层**：背景慢（呼吸感）、中景中（脉动）、前景快（响应交互）
2. **永不全屏闪烁**：霓虹明暗变化幅度限制在 70%~100%，避免视觉疲劳
3. **可交互必须暗示**：所有可点击元素 idle 时有 ≤ 5% 的微动效（浮动 / 微光）
4. **hover 反馈在 100ms 内**，点击反馈在 50ms 内，否则感觉"卡"
5. **降级策略**：低性能设备自动减少粒子数、关闭模糊滤镜，保证 30fps 底线

## 二、动态效果分层设计

按 Canvas 渲染的 **Z 轴层级** 自下而上规划，每层职责清晰：

### 🎨 Layer 0｜背景层（最慢，营造氛围）

| 元素                               | 动态方案            | 参数建议                   |
| ---------------------------------- | ------------------- | -------------------------- |
| **赛博空间背景**                   | 缓慢横向视差流动    | 0.2px/frame                |
| **代码雨（左上角 Python 代码区）** | 字符逐行高亮、滚动  | 每 800ms 高亮一行          |
| **数据流粒子**                     | 紫/青色光点缓慢漂浮 | 30~50 个粒子，速度 0.3~0.8 |
| **远处霓虹光晕**                   | 大范围低频呼吸      | 周期 4s，透明度 0.6~0.9    |

**实现要点**：用 `OffscreenCanvas` 预渲染静态背景，只在上层叠加动态粒子，性能省一半。

---

### 💡 Layer 1｜霓虹招牌层（中频，建立场域）

| 招牌                             | 动效                                | 触发逻辑                              |
| -------------------------------- | ----------------------------------- | ------------------------------------- |
| **「AI算命」主标题**             | 持续脉动 + 偶发"故障闪烁"（glitch） | 每 8~12s 随机 glitch 一次，持续 200ms |
| **「赛博玄学」「人工智能看相」** | 错峰呼吸（左 0s / 右 1s 起跳）      | 周期 3s                               |
| **「今日运势」「八字精批」**     | 字符逐个点亮（启动时）              | 入场动画，进入后保持常亮+微闪         |
| **「未来可期」**                 | 偶尔字符抖动一下                    | 每 15s 一次彩蛋                       |
| **「上上签」红色招牌**           | 暖色脉动（区别于冷色调）            | 周期 2s，emoji 加分项                 |

**关键技术**：

- 用 `shadowBlur` + `shadowColor` 做霓虹辉光，但**全图 shadowBlur 实例不超过 8 个**（性能杀手）
- glitch 效果 = 横向切片偏移 + RGB 通道分离，持续时间务必短（< 250ms）
- 启动入场建议错峰 50~150ms 点亮，形成"扫过去"的开机感

---

### 🔮 Layer 2｜中景互动元素层（重点）

#### ▶ 水晶球（核心 C 位）

**Idle 状态**：

- 内部紫色光雾旋转（`conic-gradient` + 旋转）
- 表面高光缓慢移动（模拟玻璃反射）
- 底座向上发出粒子流（5~10 个/秒）
- 「测算中」文字呼吸

**Hover 状态**：

- 光雾加速 1.5×
- 整体放大 1.05×，光晕外扩
- 鼠标拖动时，球内"光雾"跟随鼠标方向偏移（视差感）

**点击（核心交互！）**：

1. 光雾爆发性旋转（2~3s）
2. 周围 5 张塔罗牌依次飞向水晶球上方排列
3. 球内出现文字渐显："正在解读你的命运..."
4. 完成后随机一张牌"被选中"翻转，弹出占卜结果面板

#### ▶ 塔罗牌组（中央 5 张）

**Idle**：

- 上下浮动（错峰，sin 波，振幅 4px，周期 3s）
- 卡牌轻微左右摇摆（±2°）
- 牌面图案（太阳、星等）有微光描边

**Hover 单张**：

- 该牌升起 12px、放大 1.1×、辉光增强
- 其余牌**轻微暗化 + 后退**（聚焦感）
- 鼠标 3D 倾斜跟随（CSS `perspective` 或 Canvas 矩阵变换）

**点击**：翻转 → 显示牌意 → 二次点击收回

#### ▶ 八卦阴阳鱼（左侧）

**Idle**：缓慢顺时针旋转，周期 20s（不能太快，会晕）

**Hover**：旋转加速到 8s 周期，发出金色描边

**点击**：弹出"八字输入"小面板（生辰八字测算入口）

#### ▶ 手相图（右上）

**Idle**：三条线（命运线/智慧线/感情线）依次点亮循环（呼吸式）

**Hover**：线条全部点亮 + 出现"扫描线"上下移动

**点击**：触发"上传手掌照片"或"AI模拟手相分析"流程

---

### 🐱 Layer 3｜角色与道具层（萌点 + 彩蛋）

#### ▶ 中央少女（占卜师）

| 部位         | 动效                       |
| ------------ | -------------------------- |
| 眼睛         | 每 4~7s 眨一次（随机间隔） |
| 头发         | 极轻微飘动（顶点扰动）     |
| 头饰光环     | 持续旋转，周期 6s          |
| 左手前方光屏 | 显示"测算分析中"滚动文本   |
| 整体         | 呼吸式上下浮动 ±2px        |

**点击身体**：触发"占卜师寄语"气泡（增加陪伴感）

#### ▶ 赛博猫（左下）

**Idle**：

- 尾巴轻摆
- 眼睛偶尔变成 `>_<` 表情（每 10s 随机一次）
- 颈圈灯珠循环闪烁（彩虹色）

**Hover**：歪头 + 喵叫音效（可选，需用户授权）

**点击**：抽签！上方"上上签"竹筒晃动 → 飞出一支签 → 展示签文

#### ▶ 三个签筒（姻缘 / 事业 / 财运）

**Idle**：紫色光柱从筒内升起（呼吸感）

**Hover**：对应文字放大 + 光柱变亮 + 显示 tooltip

**点击**：进入对应专项占卜（这是 **3 个独立功能入口**）

#### ▶ AI 诗签卷轴（右下）

**Idle**：卷轴边缘有金色流光循环

**Hover**：展开一点，显示"点击展开完整诗签"

**点击**：卷轴展开动画 → AI 实时生成一首四句诗

#### ▶ 机械臂

**Idle**：缓慢呼吸（关节微动）

**触发占卜时**：机械臂向水晶球伸出 → 触碰 → 引发能量波纹（戏剧化的"启动"动作）

---

### ✨ Layer 4｜粒子与特效层（最上层）

- **常驻**：紫色光尘缓慢飘落，全屏 80~120 个粒子
- **占卜时**：水晶球向四周爆发粒子（200+，持续 1.5s）
- **结果揭晓**：金色星光从结果面板向外辐射
- **鼠标拖尾**（可选）：紫色光点跟随，0.3s 衰减

| 4   | 🏮 姻缘签筒      | 点击     | 感情运势                   | P1     |
| 5   | 💼 事业签筒      | 点击     | 事业运势                   | P1     |
| 6   | 💰 财运签筒      | 点击     | 财运分析                   | P1     |
| 7   | ☯️ 八卦图        | 点击     | 八字测算（需输入生辰）     | P1     |
| 8   | ✋ 手相图        | 点击     | 手相分析（上传照片）       | P2     |

**隐藏彩蛋**：

- 连续点击占卜师 5 次 → 解锁"占卜师本命"
- 凌晨 0~3 点访问 → 整体色调变冷 + 出现"夜半凶兆"特殊事件
- 连点猫 10 次 → 猫翻肚皮

### 交互反馈规范

```
hover    : 光晕 + 光标变 pointer + tooltip（300ms 延迟出现）
press    : 缩小 0.95× + 内发光（50ms）
release  : 弹回 1.0× + 涟漪扩散
disabled : 灰度 70% + 不响应
loading  : 元素被锁定，水晶球承担 loading 动画
```

## 四、技术实现建议

### 4.1 渲染架构

```
┌─────────────────────────────────┐
│  UI 层 (HTML/CSS)                │ ← 结果弹窗、表单输入
├─────────────────────────────────┤
│  Canvas 主层 (前景动画 + 交互)    │ ← requestAnimationFrame
├─────────────────────────────────┤
│  Canvas 背景层 (慢动效)           │ ← 低频更新，10fps 即可
├─────────────────────────────────┤
│  OffscreenCanvas (静态背景预渲染) │ ← 一次绘制
└─────────────────────────────────┘
```

### 4.2 推荐技术栈

- **核心**：原生 Canvas 2D（够用）或 **PixelJS / PixiJS**（更省心，自带场景图与交互系统，强烈推荐）
- **粒子**：自写简单粒子系统，或 `tsParticles`
- **物理感**：弹簧动画用 `popmotion` 或自实现 spring easing
- **命中检测**：每个可交互元素维护 `bbox`，鼠标移动时遍历（< 20 个元素无性能问题）
- **音效**（强烈建议）：`Howler.js`，水晶球启动音、卡牌翻转音、签筒摇晃音

### 4.3 性能预算

| 指标            | 目标   | 底线                     |
| --------------- | ------ | ------------------------ |
| 帧率            | 60fps  | 30fps                    |
| 首屏加载        | < 2s   | < 4s                     |
| 粒子总数        | 150    | 300（高配）/ 50（低配）  |
| 同时 shadowBlur | ≤ 8 个 | 用 sprite 预渲染辉光替代 |

### 4.4 降级策略

```js
if (低端设备 || prefers-reduced-motion) {
  关闭粒子系统;
  关闭 glitch 效果;
  霓虹改为静态 PNG + CSS 简单 opacity 动画;
  保留核心点击交互;
}
```

**无障碍**：尊重 `prefers-reduced-motion`，提供"减少动效"开关。

## 七、深度补充 · 三个关键设计的执行手册

> 前面提到的"节奏分层""戏剧化""好彩蛋"都是容易说漂亮话但难落地的概念。本节把它们变成**可量化、可验收**的工程标准。

### 7.1 节奏分层 · 怎么落地？

#### 🎯 核心原则："心跳，而非警报"

人体感觉舒服的视觉节奏来自于**多频率的错峰叠加**——像音乐的低音、中音、高音和谐共存。如果所有元素都同频跳动，大脑会识别为"警报"而非"生命"。

#### 频率分层表（直接抄作业）

| 层级               | 周期      | 频率    | 用在哪               | Easing       |
| ------------------ | --------- | ------- | -------------------- | ------------ |
| **L0 极慢层**      | 6~10s     | ~0.15Hz | 远景视差、大范围光晕 | sine         |
| **L1 慢层**        | 4s        | 0.25Hz  | 主背景呼吸、人物浮动 | sine         |
| **L2 中层**        | 3s        | 0.33Hz  | 霓虹脉动、卡牌浮动   | sine         |
| **L3 快层**        | 2s        | 0.5Hz   | 水晶球光雾、签筒光柱 | sine         |
| **L4 极快/事件层** | 200~500ms | 单次    | 点击反馈、glitch     | cubic-bezier |
| **L5 随机层**      | 不定      | 稀疏    | 眨眼、glitch、彩蛋   | step         |

#### 错峰公式

> 同层级内的多个元素，**不要让它们 phase = 0 同步起跳**。

```js
// 反例（同步，像心律不齐警报）
neon1.phase = 0;
neon2.phase = 0;
neon3.phase = 0;

// 正例（错峰，像心跳）
const elements = [neon1, neon2, neon3];
elements.forEach((el, i) => {
  el.phase = (i / elements.length) * Math.PI * 2; // 均匀错开
  // 或加随机扰动
  el.phase += (Math.random() - 0.5) * 0.4;
});
```

#### 节奏可视化（设想 5 秒内屏幕活动）

```
时间(s)  0    1    2    3    4    5
─────────────────────────────────────
背景 L1  ▁▂▃▄▅▆▇█▇▆▅▄▃▂▁▂▃▄▅▆▇█▇▆▅▄▃   ← 一次完整呼吸
霓虹 L2  ▃▅▇▅▃▅▇▅▃▅▇▅▃▅▇▅▃▅▇▅▃▅▇▅▃▅▇   ← 1.6 个周期
水晶 L3  ▅▇▅▃▅▇▅▃▅▇▅▃▅▇▅▃▅▇▅▃▅▇▅▃▅▇▅   ← 2.5 个周期
卡牌 L2' ▇▅▃▅▇▅▃▅▇▅▃▅▇▅▃▅▇▅▃▅▇▅▃▅▇▅▃   ← L2 同频但相位 +π
粒子     ····························   ← 持续匀速
眨眼     ········*··············*····   ← 随机点
```

任意时刻横切一刀，**亮度差始终存在**——这就是"活的"感觉。如果某一时刻所有元素都到峰值，立刻调相位。

#### 振幅分层（同样重要，常被忽略）

频率错开还不够，**亮度变化的幅度**也要分层，否则边缘元素抢主角戏。

| 区域                     | 振幅范围 | 含义             |
| ------------------------ | -------- | ---------------- |
| 屏幕**中心**（水晶球区） | 80%~100% | 最亮、变化最显著 |
| 中景（卡牌、占卜师）     | 70%~95%  | 次亮             |
| 边缘（招牌、远景）       | 60%~85%  | 衬托             |
| 背景（代码雨、粒子）     | 30%~60%  | 不抢戏           |

> **铁律**：永远不允许边缘元素的峰值亮度超过中心元素的谷值亮度。

#### 自检方法

1. **截图法**：录 10 秒视频，每 0.5s 抽一帧拼成 20 张图，肉眼看亮度分布是否始终主次分明。
2. **眯眼法**：眯起眼睛看页面（模糊化），主次焦点是否清晰？如果眯眼后看不出"哪里是主角"——节奏失败。
3. **静音视频法**：把动画录下来无声播放给朋友，问 ta"哪里最想点？"——如果答案不是水晶球，节奏失败。

### 7.2 戏剧化 · 怎么落地？

#### 🎯 核心原则："四幕结构 + 屏息瞬间"

任何"仪式感"的体验都遵循 **Setup → Build-up → Climax → Resolution** 四幕，缺一不可。占卜的爽点 90% 来自**高潮前那个 200ms 的停顿**——所有动画凝滞，让用户屏住呼吸的瞬间。

#### 四幕结构通用模板

| 幕                       | 时长占比 | 心理目标               | 视觉手法                   |
| ------------------------ | -------- | ---------------------- | -------------------------- |
| **① Setup（预备）**      | 5%~10%   | "我的操作被接收了"     | 即时反馈：缩放、波纹、音效 |
| **② Build-up（酝酿）**   | 50%~60%  | "事情正在发生，期待中" | 加速、聚集、能量积累       |
| **③ Climax（爆发）**     | 5%~10%   | "！"                   | 凝滞 → 释放，最高对比度    |
| **④ Resolution（回落）** | 25%~35%  | "原来如此"             | 减速、铺陈结果、可读       |

#### 工作示例：水晶球占卜帧级时间轴

```
══════ ① Setup（0~300ms，"接收"）══════
T+0      用户点击 → 立即响应
T+50     球体内陷 scale 1.0 → 0.95（物理反馈）
T+100    回弹 1.0× + 表面波纹扩散（涟漪音效）
T+200    背景音乐渐弱 30%，环境 hum 渐起
         👉 用户已知道"开始了"

══════ ② Build-up（300~2200ms，"酝酿"）══════
T+300    光雾旋转加速 1× → 3×
T+500    底座粒子发射量 ×3，颜色由紫转金
T+800    5 张塔罗牌脱离原位，被"吸引"飞向球上方
T+1200   牌阵到达，开始环绕旋转（仪式感峰值前奏）
T+1500   环境光开始向中心收缩（其他区域变暗）
T+1800   机械臂从画面右下伸出
T+2000   机械臂关节有"思考"般的微调动作
T+2200   机械臂触碰球面 → 能量波纹爆发
         👉 用户已被吊足胃口

══════ ③ Climax（2200~2700ms，"屏息"）══════
T+2200   ⭐⭐⭐ 关键：所有动画凝滞 200ms ⭐⭐⭐
         所有粒子停在空中
         音乐戛然而止
         屏幕中心一个白色闪光（80ms 内进出）
T+2400   一张牌被高光选中，其余瞬间暗化到 20%
T+2500   选中牌脱离牌阵，飞向画面中央
         👉 此刻用户脑内："就是这张了！"

══════ ④ Resolution（2700~4500ms，"揭晓"）══════
T+2700   牌开始翻转（300ms，先减速到中点再加速）
T+3000   牌面亮起，光晕外扩
T+3200   结果文字逐字浮现（typing 效果，每字 60ms）
T+3700   金色星光从牌面辐射（庆祝感）
T+4200   背景音乐重新淡入
T+4500   "再算一次 / 分享" 按钮淡入
         👉 用户已经截图了
```

#### 戏剧化的六条铁律

1. **暂停 = 力量**。Climax 前的 200ms 凝滞是整个流程最重要的 200ms，**不要省**。
2. **AI 思考时间 2~4 秒最佳**。< 1.5s 显得假（没思考），> 5s 显得笨（在卡顿）。
3. **聚光灯效应**。越接近 Climax，画面其他区域亮度越低，焦点越收窄。
4. **声音是免费的戏剧倍增器**：低频 hum 走 Build-up，clear bell 卡 Climax，弦乐 Resolution。预算允许必加。
5. **结果分至少两步揭晓**：选中 → 翻开 → 文字浮现。一步到位 = 廉价。
6. **失败也要有戏**：网络错误不是 toast 报错，而是"水晶球能量不足"+ 雾散 + 机械臂收回。

#### 戏剧化反例（避雷）

| 反例                      | 为什么糟糕                     |
| ------------------------- | ------------------------------ |
| 点击立即出结果            | 缺整个 Build-up + Climax，廉价 |
| 转圈 loading 5 秒后弹文字 | 有等待无戏剧，无聊             |
| 全程平稳节奏              | 缺 Climax 的对比度，没爽点     |
| 结果一次性全部出现        | 缺 Resolution 的层次，无回味   |
| Build-up 用进度条         | 进度条把"神秘"杀死了           |

#### 自检方法

> **静音录屏播放**：录占卜全过程的视频，**关掉声音**，给一个没玩过的人看。问 ta：
>
> - "你能感觉到一个'开始-紧张-高潮-结束'的弧线吗？"
> - "哪一秒最紧张？"（应该指向 Climax 的凝滞瞬间）
> - "结果出现时你有'啊'的感觉吗？"
>
> 三个问题都"是"，戏剧化达标。

### 7.3 彩蛋 · 怎么算"好"？

#### 🎯 核心原则："被偶遇的惊喜，而非被找到的任务"

好彩蛋的本质是用户在心流中**"撞见"**意外的反馈，得到一种"原来你看到了我"的暧昧感。一旦它变成"完成 5 个隐藏点击解锁"——它就不再是彩蛋，是任务。

#### 《彩蛋六维评估表》

| 维度                         | 1 分                             | 3 分       | 5 分                 |
| ---------------------------- | -------------------------------- | ---------- | -------------------- |
| **发现路径 Discoverability** | 无任何暗示                       | 偶有迹象   | 自然心流中能撞见     |
| **触发难度 Difficulty**      | 太易（不珍贵）或太难（无人触发） | 偏易/偏难  | 适度，需多花一点心思 |
| **奖励力度 Reward**          | 反馈微小                         | 有趣但短   | 截图/讲给朋友的冲动  |
| **主题契合 Thematic fit**    | 破坏世界观                       | 不违和     | 强化沉浸             |
| **传播性 Shareability**      | 无人会传                         | 朋友间会聊 | 社交媒体可传播       |
| **安全性 Safety**            | 影响主流程                       | 偶有干扰   | 完全独立、无副作用   |

**总分 30。 ≥ 24 = 优秀彩蛋；18~23 = 可上线；< 18 = 重做或砍掉。**

#### 用评估表回审 spec 里的彩蛋

| 彩蛋                        | 发现 | 难度 | 奖励 | 契合 | 传播 | 安全 | 总分   | 结论                    |
| --------------------------- | ---- | ---- | ---- | ---- | ---- | ---- | ------ | ----------------------- |
| 凌晨访问变冷色调 + 夜半凶兆 | 4    | 4    | 4    | 5    | 5    | 5    | **27** | ✅ 优秀                 |
| 连点猫 10 次翻肚皮          | 4    | 3    | 3    | 5    | 4    | 5    | **24** | ✅ 优秀                 |
| 连点占卜师 5 次解锁本命     | 2    | 3    | 3    | 4    | 3    | 4    | **19** | ⚠️ 改造                 |
| 鼠标拖水晶球雾跟随          | 5    | 5    | 2    | 5    | 2    | 5    | **24** | ✅ 优秀（作为 L1 彩蛋） |

> **占卜师 5 连点**之所以分低：发现路径不自然（谁会无故狂点一个角色？）。**改造方案**：改为"连续 3 天访问"解锁，理由——多次访问的用户已经与占卜师建立"羁绊"，奖励触发时机才合情合理。改造后 Discoverability 4 / Thematic 5 / Shareability 4 → 总分跃升至 25。

#### 五层彩蛋梯度（设计组合拳，而非孤立彩蛋）

好的彩蛋系统是**金字塔结构**——浅层人人都能撞见，深层只有铁粉知道：

| 层级            | 触发难度               | 受众占比  | 例子                                    |
| --------------- | ---------------------- | --------- | --------------------------------------- |
| **L1 心流彩蛋** | 0 努力，自然心流中触发 | ~80% 用户 | 拖动水晶球内雾偏移、鼠标移过卡牌时倾斜  |
| **L2 好奇彩蛋** | 主动探索一次即可       | ~40% 用户 | 点击占卜师 → 害羞表情、点击猫 → 喵叫    |
| **L3 时段彩蛋** | 特定条件被动触发       | ~15% 用户 | 凌晨访问、生日访问、节气访问            |
| **L4 羁绊彩蛋** | 多次回访               | ~3% 用户  | 连续 3 天 / 累计 7 次占卜解锁内容       |
| **L5 圈内彩蛋** | 几乎无人独立发现       | < 0.5%    | Konami code、特定八字（如制作团队生日） |

> **比例建议**：L1 占动效总数 30%，L2 占 30%，L3 占 20%，L4 占 15%，L5 占 5%。**L1 一定要多**——它们才是"这个产品有灵气"的关键来源。

#### 彩蛋反例（避雷）

| 反例                                        | 病灶                                             |
| ------------------------------------------- | ------------------------------------------------ |
| ❌ "找齐 5 个隐藏点击解锁"                  | 任务化，杀死惊喜感                               |
| ❌ 触发后影响主功能（解锁后水晶球反而变样） | 安全性失败                                       |
| ❌ 反复触发同样反馈                         | 应有 cooldown 或递进版本                         |
| ❌ 弹出"恭喜你发现彩蛋！"                   | 这是成就感设计，不是彩蛋设计——彩蛋应保持**暧昧** |
| ❌ 彩蛋打破世界观（赛博占卜里弹公司 logo）  | 主题契合 1 分，直接砍                            |

#### 终极自检问题

设计每个彩蛋时，强迫自己回答：

1. **用户在什么"心流瞬间"会自然触发它？** ——答不出来 = 彩蛋路径不自然
2. **触发后用户的第一反应是想分享、想再玩、还是 OK 关掉？** ——只有"OK 关掉" = 奖励不够
3. **如果这个彩蛋永远没人发现，产品损失什么？** ——什么都不损失 = 砍掉
4. **如果这个彩蛋全网都知道了，会破坏体验吗？** ——会破坏 = 奖励过度，会变成"必须打卡"

---

> **最后一句**：技术上能做的动效有 100 种，但每加一个都要问自己——"它让用户更想留下，还是更分散注意力？" 占卜的核心是**人对未知的期待**，所有动效都是为了放大这份期待感，而非炫技。
>
> 节奏让产品**像活的**，戏剧让产品**有魂**，彩蛋让产品**有人味**。三者缺一，做出来的就只是一张会动的图。
